# Requirements

## Functional Requirements

The project shall:

- Import multiple public datasets, combining them for the final database.
- Store data within a relational SQL database.
- Allow querying of game market information.
- Analyse historical market trends.
- Compare games based on characteristics.
- Produce statistical summaries.
- Generate an interactive Power BI dashboard.

---

## Non-Functional Requirements

### Data Quality

- Data should be cleaned before analysis, including duplicate and missing values.

### Reproducibility

- Python scripts should reproduce all analyses.
- SQL scripts should recreate the database.

### Documentation

- All datasets should be documented.
- Analysis workflow should be reproducible.

### Performance

- SQL queries and Power BI visuals should run efficiently without lag.

---

## Success Criteria

- Database has been implemented.
- Data has been cleaned.
- Key business questions have been answered.
- Scenario analysis and forecasting is possible.
- Dashboard is complete.
- Documentation has been finalised.
